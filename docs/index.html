为什么需要它

把「大模型调用」这一层单独抽出来，渲染等重活仍在老师本机

不用它的时候

  • 整套 AI 应用部署在校服务器，多人用就排队、算力扛不住
  • 让每位老师自己买 API，又麻烦又费钱
  • 真实厂家 Key 在老师手里，难管理、易泄露

用了 OpenKeyHub

  • 网关只转发请求、几乎不耗资源，一台普通机器扛全校、不排队
  • 学校统一配 Key，老师零成本、零门槛
  • 真实 Key 只在服务端、加密存储，老师只拿到网关 Key

核心特性

窄而轻 —— 只做「填厂家 Key → 发老师 Key → 看用量」三件事

🔌

填名即转发

OpenAI 兼容的 /v1/chat/completions，含流式。老师在任意客户端填模型名，网关按名自动匹配厂家。

🔑

老师自助取 Key

手机号 + 默认密码登录老师端，首登强制改密后自助查看自己的 Base URL 和 Key，管理员无需逐个分发。

📊

用量看得见

每位老师的调用次数、按模型汇总、token 用量、调用日志，一目了然；可选限额 / 限速。

🪶

轻量本地

FastAPI + SQLite，4 个依赖，一条命令起；数据全在本地、自主可控；提供 Windows 免安装包。

三步用起来

管理员配置一次，老师此后零打扰

配厂家

后台选预设（DeepSeek / 通义 / 智谱…），粘上真实 API Key，勾选要开放的模型。

发老师

单建或 姓名,手机号,备注 批量导入，每人生成一个网关 Key；老师也可自助登录领取。

填进客户端

老师在 AI 客户端选「OpenAI 兼容」，填 服务地址/v1 + 个人 Key + 模型名，即用。

内置国内厂家预设

一键填地址，只需粘 Key
